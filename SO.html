Il kernel

il kernel Parte da un nucleo centrale ovvero il kernel, il kernel è il tramite tra la parte hardware e la parte software, si occupa di gestire il processore e risale pian piano verso l’esterno affinché l’utente può avviare i programmi

Il sistema operativo

il sistema operativo è un interfaccia che collega gli utenti e il dispositivo mettendo a disposizione strumenti di lavoro e piccole procedure che facilitano L’interazion

L'avvio dei dispositivi

  1. Comincia con la parte di boot dove ci sono due accensioni, una fisica che prevede l'alimentazione di tutti i componenti e una logica (inizializzazione)
  2. Il processore carica il BIOS (Basic Input Output System) che si occupa delle routine successive (Di solito è incorporato col UEFI)
  3. Successivamente il BIOS procede alla fase di diagnostica detta POST (Power On Self Test) che verifica il corretto funzionamento di tutte le componenti
  4. Se la fase POST viene superata il BIOS carica dalla memoria il Boot Loader ovvero il programma dedicato all avvio dell SO anche se non carica tutto il sistema operativo ma soltanto il kernel
  5. Terminata anche la fase di boot il PC passa sotto il controllo del sistema operativo di log dell utente (login) per accedere al Desktop

Il Desktop

quasi tutti i SO adottano Un interfaccia grafica (GUI) Touch friendly, che non solo è a portata di mouse e tastiera ma anche di tocco (multi touch), ma esistono anche SO con interfaccia a riga di comando (come ms - dos)

iL Desktop èformato da:

  • Il desktop: è l’ambiente di lavoro principale, come dice il nome “Desktop” è come una scrivania con cui l’utente può inserire nuove icone e aprire programmi
  • Desktop virtuali: inseriti negli ultimi sistemi operativi servono ad avere diversi desktop allo stesso momento per evitare confusione, dove per esempio in uno ci lavori con email e in un altro ci fai le cose personale come viaggi o social media
  • Le icone: sono degli elementi grafici che identificano programmi,file,risorse ecc.. (come la F di Facebook)
  • Le App: Attraverso le icone si possono aprire le applicazioni specifiche
  • Widget: sono delle piccole applicazioni che permettono di accedere a delle informazioni piu velocemente e facilmente mostrando solo una piccola parte (come il widget del meteo)

Il firmware

Il pc parte con il firmware che esegue delle routine per l’avvio del pc, prima si trovava nella ROM ora nella NVRAM (di tipo flash)

I suoi compiti sono:

  • controllare l’avvio del pc
  • inizializzare le periferiche
  • ricercare i drive su cui è installato il SO
  • avviare le funzioni di base del kernel del SO

Il BIOS e il UEFI

Il bios è il firmware introdotto nei primi pc poi superato dal UEFI (Unified Extensible Firmware Interface)

il bios svolge funzioni di basso livello in grado di accedere all hardware

bios più diffusi sono ami bios, phoenix bios e award bios

il bios pure può essere aggiornato e questo processo si chiama flashing BIOS (solitamente per riparare errori)

questo aggiornamento non si effettua spesso, in caso l’alimentazione venga staccata durante l’aggiornamento può causare gravi danni alla scheda madre (brick)

La batteria CMOS

La complementary Metal Oxide Semiconductor (CMOS) era una piccola memoria nella scheda madre che memorizza e impostazioni del bios, nelle schede madri attuali sono memorizzate nella in una memoria volatile della RAM (NVRAM)

Il Bootstrap

Il bootstrap del PC è il processo attraverso il quale il computer carica e avvia il sistema operativo, caricando il codice di avvio iniziale (detto bootloader) dal firmware del computer (BIOS o UEFI) e poi eseguendo il codice del bootloader per avviare il sistema operativo presente sul disco rigido o su altri dispositivi di archiviazione.

Il Master boot record

Il bios non conoscendo l’organizzazione dei file può leggere solo blocchi del disco, di conseguenza il boot loader è salvato nel primo hard disk avviabile, e occupa 512 byte chiamato Master Boot Record (MBR) (Nei sistemi linux si chiama slave boot program)

Nei 512 byte dell MBR:

  • i primi 466 contengono il first stage boot loader, un codice eseguibile scritto in assembly che scansiona i boot sector per trovare il second stage boot loader
  • i successivi 64 hanno la partition table che contengono le informazioni di partizione del disco
  • gli ultimi 2 contengono il magic number al quale si fa riferimento il termine signature che è usato per indicare la fine del MBR

File Systems e Struttura partizioni

Il file system è la struttura usata dal sistema operativo per organizzare e gestire i file in un dispositivo , Quando si formatta una partizione di deve scegliere quale file system usare,per esempio:

  • NFTS (New technology file system): utilizzato dai sistemi windows
  • FAT32 (File Allocation Table): utilizzato principalmente dai vecchi sistemi windows e linux
  • ext4 (fourth extended filesystem): utilizzato sempre dai sistemi linux
  • primaria: quella richiesta dai sistemi windows dove ci possono essere max 4 partizioni primarie
  • estesa: in un disco ce ne può essere solo una e può essere suddivisa in ulteriori unità logica

La tabella delle partizioni è strutturata in questo modo:

  • stato: 1 byte che assume come valore 0x80, quando è attiva vale 0x00
  • Primo settore CHS (Cylinder Head Sector): 3 byte che contengono l’indirizzo del primo settore della partizione
  • Tipo: 1 byte che contiene un numero che identifica il file system della partizione
  • Ultimo settore CHS: 3 byte che contengono l’indirizzo dell’ultimo settore della partizione
